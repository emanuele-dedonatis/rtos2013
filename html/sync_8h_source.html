37  * Small dirty trick.
38  * Starting from Miosix 1.58 the upgrade to the new gcc 4.5.2 and the
39  * miosix-specific patches becomes mandatory, or the kernel will no longer
40  * compile. To make the transition as smooth as possible to those who are
41  * using the old arm-miosix-eabi-gcc compiler, we need to print an
42  * informative warning telling them they have to upgrade.
43  * Now, since the new patches make the #define PTHREAD_MUTEX_RECURSIVE
44  * available we can use it to check which compiler is being used.
45  */
